Features
Was ContainerGen generiert

Nicht nur ein Dockerfile. Jeder Service liefert einen vollständigen Stack — vom lokalen Dev-Setup bis zum Kubernetes-Produktions-Deployment.

🐳
Dockerfile
Optimierter Build mit korrektem Base-Image, Layer-Caching und Multi-stage Build wo sinnvoll — z.B. bei Go oder Java.
⚙
docker-compose.yml
Vollständiges Compose-File mit Healthchecks, named Volumes, Depends-on Logik und konfigurierbaren Umgebungsvariablen via .env.
☸
Kubernetes Manifest
Deployment, Service und HPA mit SecurityContext, Resource Limits und konfigurierbarer Replica-Anzahl.
🛡
Zero-Trust Toggles
Non-root User, Read-only Filesystem, No new privileges und Cap-Drop ALL — einzeln ein- und ausschaltbar.
🚀
CI/CD Pipeline
GitHub Actions und GitLab CI Templates werden automatisch mitgeneriert — Build, Test und Deploy in einem Workflow.
✏
Direkt bearbeitbar
Jede Datei ist direkt im integrierten Editor anpassbar. Geänderte Tabs werden orange markiert, Reset ist jederzeit möglich.
# Beispiel — generiertes Dockerfile für Python Flask
FROM python:3.12-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .

# Zero-Trust: Non-root User
RUN useradd -m -u 1000 appuser && chown -R appuser:appuser /app
USER appuser

# Read-only Filesystem — tmpfs für Laufzeitdaten
EXPOSE 5000
CMD ["python", "-m", "flask", "run", "--host=0.0.0.0"]
